首先我们只有 O(n2)\mathcal O(n^2)O(n2) 个点有用，就是六个方向的光打过去的每一面的第一个点。

我们先考虑怎么求这些点，我们存一下障碍物，在每一行每一列的第一次出现和最后一次出现，然后不在这个范围内即可。类似的开一共 666 个二维数组存一存。

然后显然题意的第一类和第二类没什么区别，可以放一起做。

我们维护这样两个信息，可以很好做：点 (x,y,z)(x,y,z)(x,y,z) 在方向 ddd 上是否能被打到光；以及点点 (x,y,z)(x,y,z)(x,y,z) 在方向 ddd 上最后会坠落到哪。

第一个问题其实就是看某一方向上是否在障碍物之前。第二个问题就是掉到障碍物上。代码大概长这样：

bool ok_sun(P pos, int d)
{
    auto [x, y, z] = pos;
    if (d == 0)
        return z < zmin[x][y];
    if (d == 1)
        return zmax[x][y] < z;
    if (d == 2)
        return y < ymin[x][z];
    if (d == 3)
        return ymax[x][z] < y;
    if (d == 4)
        return x < xmin[y][z];
    if (d == 5)
        return xmax[y][z] < x;
    return 0;
}
P real_pos(P pos, int d)
{
    auto [x, y, z] = pos;
    if (d == 0)
        return {x, y, zmin[x][y] - 1};
    if (d == 1)
        return {x, y, zmax[x][y] + 1};
    if (d == 2)
        return {x, ymin[x][z] - 1, z};
    if (d == 3)
        return {x, ymax[x][z] + 1, z};
    if (d == 4)
        return {xmin[y][z] - 1, y, z};
    if (d == 5)
        return {xmax[y][z] + 1, y, z};
    return {-1, -1, -1};
}

然后考虑移动。第一个是，我们枚举 666 个方向以及算一下走过去的点，然后再枚举 666 个方向，代表光打过来的方向。如果两个点都可以被打到光，就可以走。然后通过 real_pos(pos, d) 算一下走过去会掉到哪。还有一类是，直接修改重力方向，类似的判一下 ok_sun(pos, d) 然后走到 real_pos(pos, d) 即可。

题意转化为 xxx 当根的深度唯一的点才会造成贡献。

首先有一个发现：能对 xxx 产生贡献的点，一定在以 xxx 为起点的最长链上。否则，一定能在最长链上找到一个深度相同的点。

而一个点出发最远的点，肯定是直径的某一个端点。因此我们不妨先求出直径，把直径某个端点当成根跑一下，有贡献的点只在根到自己的路径上。

考虑路径上其实有一些点不会产生贡献，就是有一些祖先，祖先往路径外的某个子树，距离很长，就把我们的路径上一些点给覆盖掉了。

为了处理这个，我们处理一个点往下最深的两条链。我们先把第二深的覆盖掉，然后 DFS 到最深的里面做。做完之后，用最深的覆盖掉，然后 DFS 到其他子树。

用一个栈维护路径上此时还有效的点即可。数种类数维护一个桶即可。
