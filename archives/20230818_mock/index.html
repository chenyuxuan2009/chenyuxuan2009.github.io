护手

没有强制在线，我只用莫队！！1

首先你会发现这道题长了一脸莫队的样子，区间询问又是可以离线的。

然后你直接拍一个莫队上去，会发现 T=0T=0T=0 奇数分是很好拿的。因为我们知道 x⊕x=0,x⊕0=xx \oplus x=0,x\oplus 0=xx⊕x=0,x⊕0=x，那么奇数次出现就是区间所有的异或起来。只拿这个分甚至可以维护前缀直接异或直接 O(1)\mathcal O(1)O(1) 回答问题。

但事实上我考场是凑了半天发现不用判断是第几次出现直接异或就可以的。

然后你考虑偶数的怎么搞，你想，区间里非重复元素的集合减去出现奇数次元素集合不就是出现偶数次元素集合吗！

所以你维护某个区间去重之后的元素的异或，把偶数当成奇数做，最后和上树的去重元素异或异或一下就好。

去重元素异或维护很容易，统计每个数出现次数，删去之后全都没了和加上之后是第一次都异或起来。得分 83 pts83\ \text{pts}83 pts。

怎么回事呢？

考虑进行卡常。

我们会发现我们 不在意每个数具体出现多少次，只在意他前面和后面出现的位置。 因为我们只需要判断是否在区间内出现过。

考虑维护 preipre_iprei​ 表示第 iii 个位置上一个与其相等元素位置，nxtinxt_inxti​ 表示第 iii 个位置下一个与其相等元素位置。

然后莫队区间移动只需要判断 pre,nxt,l,rpre,nxt,l,rpre,nxt,l,r 的大小关系了。本地测试最长点耗时 1.7 s1.7\ \text{s}1.7 s。

再插一嘴，你要是 nxt,prenxt,prenxt,pre 或者是 83 pts83\ \text{pts}83 pts 做法维护的数字出现次数的数组，用 map/unordered_map 是真的要死。map 稳定 log⁡\loglog，unordered_map 均摊常数实则会被卡到线性。

你复杂度再乘上一个 log⁡\loglog 你卡的过去？？？

#define getchar() p1 == p2 && (p2 = (p1 = buf) + fread(buf, 1, 1000000, stdin), p1 == p2) ? EOF : *p1++

inline int read()
{
	register char c = getchar();
	register int x = 0;
	while (c < '0' || c > '9')
		c = getchar();
	while (c >= '0' && c <= '9')
		x = (x << 1) + (x << 3) + (c ^ 48), c = getchar();
	return x;
}
